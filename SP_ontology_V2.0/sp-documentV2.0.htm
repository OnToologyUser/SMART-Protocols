Abstract

The document module of SMART Protocols is expressed in OWL. This ontology aims to provide a structured vocabulary of concepts to represent information for recording and reporting an experimental protocol.

The OWL encoding for Version 3.0 of the SMART Protocols-Document Module can be found here

3. SMART Protocol-Document Description back to ToC

The document module of SMART Protocols intends to be compatible with other formalisms, to share and reuse already formalized knowledge. For example it reuses classes from CHEBI, EXACT, MGDEO, SO , OBI and SNPO. Also, SMART Protocols-document reuses the OBO Relation Ontology (RO) and extending the class iao:InformationContentEntity proposed by the Information Artifact Ontology (IAO) to represent the laboratory protocol as a iao:Document that ro:hasParts iao:DocumentPart ( sp:IntroductionSection, sp:MaterialsSection and sp:MethodsSection). Each one of these sections includes information from a checklist to report an experimental protocol. This checklist is the result of analyzing 175 experimental protocols related to plant biology and validating it with 20 domain experts (Figure 1). In addition to maximizing the reuse of relevant domain ontologies and according to the OBO Foundry best practices, each term from SMART Protocols-document is represented by annotation properties imported from OBI_Minimal_metadata.

Figure 1: The document aspects in a protocol are captured with IAO. The terms proposed in SMART Protocols use the sp prefix.

4. Use Case back to ToC

The document module of SMART Protocols represents information such as, the protocol type, sp:DNAExtractionProtocol; it has a tittle, identified by the property sp:hasTitle, it is instantiated by Genomic DNA Isolation. Also, the author entry, iao:author identification, is instantiated by CIMMYT. This protocol is derived, sp:ProvenanceOfTheProtocol, from the protocol published by Saghai-Maroof et al., 1984 (PNAS 81:8014-8019) and its purpose is instantiated by Plant DNA Extraction Of High Quality (Figure 2).
